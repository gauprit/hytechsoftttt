Privacy Policy

Last updated: January 2025 — This policy explains how Hytechsoft (managed by Hyquantix Technologies) handles data, the technical design behind our tools, and the rights available to users (GDPR, CCPA).

At a glance

  • No file uploads by default: Tools process files inside your browser — files stay on your device.
  • No tracking: We do not use tracking cookies, cross-site trackers or fingerprinting.
  • Minimal telemetry: We may collect anonymized, aggregated usage metrics and non-identifying error reports to improve the service.

1. Scope & Applicability

This policy applies to all Hytechsoft web tools and pages hosted under hytechsoft.com and related subpaths. It covers the types of information handled, how it’s processed, third-party libraries used, and user rights under privacy laws.

2. What We Do NOT Collect

We do not collect, store, or transmit the following:

  • Personal identifiers (name, email, phone, address)
  • IP addresses or persistent device identifiers
  • Uploaded document contents (PDFs, images, recorded audio/video)
  • Authentication credentials or user account data
  • Payment information

3. What We May Collect (Minimal, Non-Identifying)

To operate and improve the service we may collect:

  • Aggregated usage counts (e.g., tool X was used N times)
  • Non-identifying browser/device type (user agent family, not full UA)
  • Anonymous error logs (stack traces stripped of file content and user data)

All analytics are anonymized and cannot be used to identify individual users.

4. Client-Side Processing — Technical Details (Safe by design)

Our tools are intentionally implemented to keep user data local. Below is a conceptual overview and a short code example showing the typical pattern used by our PDF and QR tools.

Design principles

  • Local file access: Files are read using `File` APIs and `ArrayBuffer`/`Blob` in the browser.
  • In-memory processing: PDF parsing/rendering and transformations occur in memory (Canvas, PDF.js, jsPDF, pdf-lib).
  • No automatic uploads: No file is sent to any remote server unless the user explicitly chooses an optional upload feature (which will always be communicated and require explicit consent).
  • Permissions: The browser handles permissions (file chooser, microphone), and the site does not request any unusual permissions.

Typical client-side code pattern (example)

This simplified snippet shows how we read a PDF file and render a page to a canvas (used in compressors / converters). It is the same basic approach in production, just more robust and with error handling.

// Read a PDF file and render first page to canvas (simplified)
const file = document.querySelector('#pdfInput').files[0];
if (!file) throw new Error('No file selected');

const arrayBuffer = await file.arrayBuffer();
const loadingTask = pdfjsLib.getDocument({ data: arrayBuffer });
const pdf = await loadingTask.promise;
const page = await pdf.getPage(1);
const viewport = page.getViewport({ scale: 1.0 });

const canvas = document.createElement('canvas');
canvas.width = Math.round(viewport.width);
canvas.height = Math.round(viewport.height);
await page.render({ canvasContext: canvas.getContext('2d'), viewport }).promise;

// Now canvas contains rendered page pixels; convert to image or process further.
// No data was sent to any server.
const imgData = canvas.toDataURL('image/jpeg', 0.8);

5. Optional Server Features (explicit consent required)

Some advanced features (for example: OCR for scanned documents, server-speed large-file processing, or optional cloud backup) require the file to be uploaded. These features are not enabled by default. If implemented they will:

  • be opted-in explicitly by the user
  • show a clear consent dialog describing exactly what will be uploaded
  • use encrypted transport (HTTPS) and be retained only as described

6. Third-Party Libraries & CDNs we use

We rely on well-known client libraries to perform in-browser functionality. Examples include:

  • pdf.js — PDF parsing & rendering
  • pdf-lib / jsPDF — PDF create/write
  • SheetJS (xlsx) — Excel export for PDF→Excel
  • Tailwind CSS — UI styles (via CDN)

These libraries are run inside the browser and do not, by themselves, receive your files. We keep library usage to the minimum necessary and monitor for security updates.

7. Cookies & Similar Technologies

We do not use tracking cookies or third-party advertising cookies. Minimal session cookies or localStorage may be used for UI preferences only (for example: selected theme). You can clear these from your browser at any time.

8. Security Measures

We take the following steps to protect service integrity:

  • Serve all pages over HTTPS
  • Use secure CDNs and keep libs up-to-date
  • Minimize privilege and avoid storing files on server
  • Sanitize user-provided filenames before download
  • Limit browser-side memory usage and provide graceful failure messages

9. Data Retention & Logs

We retain only anonymized, aggregated metrics and short-lived error logs for debugging. These logs do not include file contents or personally identifying information and are automatically deleted after a retention period (typically 30–90 days).

10. Children’s Privacy

Our services are not designed for children under 16. We do not knowingly collect information from children. If you believe a child under 16 has provided information, contact us to request deletion.

11. Your Rights (GDPR & CCPA)

GDPR (EU users) — Because we do not collect personal data by default, most GDPR data subject requests are not applicable. If you have used an optional server-upload feature, you can request access, rectification, portability, or erasure of your uploaded data by contacting us.

CCPA (California residents) — You may request information about categories of personal data collected, and opt-out of sale. We do not sell personal data and do not create user profiles for marketing.
